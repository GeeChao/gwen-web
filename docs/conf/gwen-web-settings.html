Gwen-Web Settings

< Gwen-Web Wiki
The following settings can be used to configure certain runtime aspects. These can be defined in your local gwen.properties file (you will need to create it in your user home directory if it does not exist) or in a separate properties file passed to gwen at launch time through the -p|--properties command line option.

Click on a setting to reveal details..

Since v1.0.0

gwen.feature.failfast

Enables or disables fail fast mode at the feature level. If enabled, will fail a feature as soon as the first sceanrio in that feature fails before resuming with the next feature (if provided).
  • default: true
  • supported: true to enable, false to disable

gwen.feature.failfast.exit

If enabled, will exit execution as soon as any feature step fails.
  • default: false
  • supported: true to enable, false to disable

gwen.report.slideshow.framespersecond

The default number of frames per second (speed) in generated slideshow
  • default: 1
  • supported: A number represeting the desired number of frames per second

gwen.report.overwrite

Overwrites any previously generated report if enabled or creates a timestamped backup otherwise
  • default: false
  • supported: true to enable, false to disable

gwen.web.browser

Configures the browser that Gwen will use
  • default: firefox
  • supported: firefox, chrome, safari, or ie

gwen.web.useragent

Set the user agent header in the browser to the literal specified
  • supported: A string literal represeting the header you want to set

gwen.web.authorize.plugins

Controls whether or not to allow browser plugins to run (for example: Java applets)
  • default: false
  • supported: true to enable, false to disable

gwen.web.wait.seconds

Sets the maximum number of seconds the web driver should wait for web elements to become available before timing out
  • default: 10
  • supported: A number of seconds representing the desired timeout period

gwen.web.maximize

Controls whether or not the web driver should maximize the browser window
  • default: false
  • supported: true to maximise, false to minimise

gwen.web.throttle.msecs

Sets the number of milliseconds to wait for the web driver and web browser to synchronize with each other between interactions. Also directly controls element highlighting duration.
  • default: 200
  • supported: A number representing the desired throttle speed in milliseconds

gwen.web.highlight.style

Sets the HTML style used by the element highlighting mechanism
  • default: background: yellow; border: 2px solid gold;
  • supported: A HTML style string

gwen.web.capture.screenshots

Controls whether or not to capture screenshots and generate slideshow
  • default: false
  • supported: true to enable, false to disable

gwen.web.capture.screenshots.highlighting

Controls whether or not to capture screenshots of element highlighting (in addition to just screenshots)
  • default: false
  • supported: true to enable, false to disable

gwen.web.remote.url

If set, allows gwen-web to connect to a remote selenium service at the specified URL.
  • supported: A URL string to a selenium grid server (http://host:port/wd/hub), saucelabs (http://:@ondemand.saucelabs.com:80/wd/hub), or other service

gwen.web.accept.untrusted.certs

Controls whether or not the web driver should accept untrusted (self signed) SSL certificates
  • default: true
  • supported: true to accept untrused SSL certs, false to deny

gwen.web.suppress.images

Controls whether or not images are rendered in the browser (suppored for firefox browser only)
  • default: false
  • supported: true to suppress image rendering, false to enable

gwen.web.chrome.extensions

Comma separated list of paths to chrome extensions (crx files or directories) to load
  • supported: Comma separated list of paths to crx files (or just a single path)

log4j.configuration

URL to log4j configuration file (example: file:/path-to-log4j-config-file. If not specified, then the log4j.properties bundled in the binary is used by default)
  • supported: Path to log4j configuration file
Since v1.3.0

gwen.rampup.interval.seconds

Sets the ramp up interval (in seconds) for staggered parallel execution (supported for parallel execution mode only). If it is not set or is set to zero, then no staggering will occur (as per default behavior). See also: Add ramp up interval for staggered parallel execution
  • supported: A number representing the ramp up interval in seconds

gwen.report.suppress.meta

Controls whether or not meta reports should be suppressed when generating HTML reports
  • default: false
  • supported: true to suppress meta reporting, false to enable
Since v1.4.0

gwen.web.capture.screenshots.duplicates

Controls whether or not contiguous duplicate screenshots should be captured or discarded
  • default: false
  • supported: true to capture duplicates, false to discard duplicates
